Cross-Site Scripting (XSS) Demo

XSS attacks inject malicious scripts into web pages. This is a safe demonstration of how input validation prevents such attacks.

How XSS Works:

1. Injection
Attacker inputs malicious script in form fields
2. Storage/Reflection
Script is stored in database or reflected in response
3. Execution
Script runs in victim's browser when page loads
4. Prevention
Sanitize input, escape output, use Content Security Policy
